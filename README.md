# APSG (Staging Ground) Report System
**Developed by Karthi** · Phase 3 · Unified Dashboard

## Features
- Daily Report (Online + WB comparison, validation, Excel download)
- Excel Rejection / Rejection Filter → PowerPoint
- PPT Alignment (Top + Front Photo Merge)
- Bulk Bundle Report Filter
- User authentication (login / register)
- Admin panel with user management & activity analytics
- **Email notifications** to admin on user activity (signup, login, module usage)

## Deployment on Render

### 1. Push to GitHub
```bash
git init
git add .
git commit -m "APSG Dashboard — production build"
git remote add origin <your-repo-url>
git push -u origin main
```

### 2. Create a new Web Service on Render
- **Build Command:** `pip install -r requirements.txt`
- **Start Command:** `bash start.sh`
- **Environment:** Python 3

### 3. Environment Variables (set in Render dashboard)
| Variable | Required | Description |
|---|---|---|
| `SECRET_KEY` | ✅ | Auto-generated by Render |
| `PORT` | ✅ | Set to `10000` |
| `ADMIN_EMAIL` | Optional | Admin email to receive notifications |
| `SMTP_HOST` | Optional | SMTP server (default: `smtp.gmail.com`) |
| `SMTP_PORT` | Optional | SMTP port (default: `587`) |
| `SMTP_USER` | Optional | Sender Gmail/email address |
| `SMTP_PASS` | Optional | Gmail App Password (not regular password) |

### Gmail App Password Setup
1. Enable 2FA on your Gmail account
2. Go to Google Account → Security → App Passwords
3. Generate a password for "Mail"
4. Use that as `SMTP_PASS`

## Email Notification Format
When a user performs an action, the admin receives:
- **User** (username)
- **Action** (LOGIN, REGISTER, OPEN_APP, DAILY_REPORT, etc.)
- **Detail** (e.g. which module was accessed)
- **IP Address**
- **Timestamp**
